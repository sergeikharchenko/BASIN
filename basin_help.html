BASIN User Guide

Bayesian Sediment Source Apportionment Tool | Documentation

BASIN is a state-of-the-art modular framework for robust sediment source fingerprinting. Built on the Stan probabilistic programming language (utilizing the Hamiltonian Monte Carlo algorithm), it overcomes traditional equifinality traps by evaluating structural geochemical covariance and dynamically correcting for non-conservative tracer behavior.

1. Input Data Format

Your basic CSV file must follow a standard structure. The first two columns are fixed as identifiers:

Col 1: ID Col 2: Group/Source Col 3...N: Variables
Sample ID (e.g., S_01) Source Category (e.g., Forest) Tracer concentrations (Fe, Zn, Al...)
... multiple source rows ...
Mix_Sample_A Target label (Mix) Concentrations in the mixture
Dynamic Column Handling: Columns selected as an SSA Proxy (e.g., Specific Surface Area) or Grouping Covariates (for hierarchical modeling) are automatically excluded from the tracer matrix to prevent mathematical redundancy.

2. Intelligent Tracer Pre-Filtering

Instead of rigid statistical exclusion that destroys multidimensional geometry, BASIN offers a 4-step intelligent pre-filtering module to safely remove strictly non-conservative tracers while retaining covariance structures:

  • Modified SSA-Aware Range Test: Dynamically checks ranges considering granulometric shifts.
  • Geochemical Association Preservation Test (GAPT): Retains tracers if they preserve at least 50% of their structural pairwise correlations (isomorphic pairs/sorption complexes) during transit.
  • PCA Convex Hull Penalty: An n-dimensional outlier detection mechanism to drop tracers pulling mixtures outside the physically robust mixing polygon.
  • KW + CORR: Standard Kruskal-Wallis discrimination test combined with a strict multicollinearity purge (cutoff > 0.95).

3. Model Core & Covariance Architecture

BASIN avoids the "independent tracer" assumption by allowing you to explicitly model geochemical structural signals. Choose one of the core error structures:

  • Independent (Fastest): Covariance matrix is strictly diagonal. Elements are treated as independent random variables (similar to classic models).
  • Mixture Covariance (Recommended): Estimates a multidimensional residual covariance structure for target mixtures via Cholesky decomposition, extracting elemental co-migration patterns directly from the ensemble.
  • Full Bayes: The most comprehensive mode, evaluating multi-dimensional tracer covariance matrices simultaneously within both the target mixtures and the source groups.

4. Advanced Computational Modules

Tailor the mathematical complexity of the dynamically compiled Stan code to your specific dataset challenges:

H Hierarchical Logic & Spatial Priors

For spatiotemporal decomposition (e.g., flood seasons or sediment core horizons), individual mixture proportions are centered on a higher-level global vector using Dirichlet distributions. Prior Geographical Weights (e.g., connectivity indices or sub-basin areas) can be explicitly integrated into the global Dirichlet hyperparameters.

B Beta-Correction (Particle Size/Sorting)

Applies a non-linear, tracer-specific exponential correction directly within the MCMC sampler to compensate for hydrodynamic sorting, tied to the standardized Z-score of the Specific Surface Area (SSA) proxy.

μcorrected = μsource · exp(βsorting · (SSAmix - SSAsource_avg))

CoDA Compositional Data Analysis (ILR)

Maps closed relative data (summing to 100%) into an open Euclidean space using an Isometric Log-Ratio (ILR) transformation. This eliminates spurious correlations inherent to closed systems. Note: Use cautiously on heavily non-conservative or hydrodynamically sorted datasets where the linear mixing assumption in log-ratio space may be violated.

Bias Robust Bayesian Bias Absorption

Introduces a random displacement vector (math sponge) utilizing a heavy-tailed Student's t-distribution. It mathematically absorbs systematic measurement errors, redox-mobility chaotic noise, or signals from unsampled "ghost" erosion hotspots, preventing them from skewing the proportions of known sources.

5. Mass-Balance & Out-of-Sample Performance

BASIN natively integrates tools to ensure both internal mathematical convergence and external physical truth:

  • Reconstruction (SMAPE): Evaluates how well the estimated source proportions mathematically reconstruct the target mixture's tracer values.
  • PSIS-LOO Cross-Validation: Pareto Smoothed Importance Sampling Leave-One-Out cross-validation. Estimates the out-of-sample predictive accuracy (ELPD) directly from posterior simulations to help you objectively select the most parsimonious model configuration without overfitting.

6. Validation (Virtual Mixtures)

To verify the model's structural accuracy, use the Validation tab to generate stochastic virtual mixtures. This respects natural source covariance (drawing from multivariate distributions rather than simple means) across the entire simplex.

Key Performance Metrics:
  • MAE: Mean Absolute Error. Check for accuracy-precision trade-offs.
  • NSE: Nash-Sutcliffe Efficiency. Evaluates overall model efficiency (>0.50 is good, >0.70 is excellent).
Download: Export the validation dataset (True vs. Predicted proportions) as a CSV for external reporting.

7. MCMC Diagnostics & Export

Always verify chain convergence before interpreting source apportionments:

Rhat (&Rcirc;): Should be strictly < 1.05. If higher, the Hamiltonian chains have not converged (increase iterations or simplify the model).

n_eff: Effective sample size. Should generally exceed 400-700 to ensure reliable posterior inferences.

Results are dynamically saved. Manual export options include:

  • Posterior Summaries: Global and individual apportionment statistics (Means, Medians, CI bounds).
  • Density Plots: High-quality visualizations of posterior probability distributions.
  • Raw MCMC Chains: Export full Stan iteration arrays for independent post-hoc auditing.
